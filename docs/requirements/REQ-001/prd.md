# REQ-001：UI 框架搭建 — PRD

## 背景与动机

"小星 Star Kids" 是一个面向 6-12 岁儿童的 AI 视觉对话桌面应用。第一里程碑是搭建完整的 UI 框架（壳子），用模拟数据验证视觉语言、交互流程和角色呈现。本 PRD 定义了壳子的完整视觉规格。

## 核心设计方向

**一句话**：星露谷物语 × AI 伙伴。手绘童话感、木屋温馨氛围、拒绝科技感。

**关键否定**：
- ❌ 不要科技蓝、不要赛博风、不要高饱和荧光色、不要纯黑
- ❌ 不要太幼稚（不是低幼卡通），而是"高端独立游戏美术质感"
- ❌ 不要冷淡/疏离——这是一个有情感温度的陪伴型产品

## 一、顶部摄像头视频区（55%）

### 框架
- **形状**：窗户式样，圆角（border-radius: 20-24px），木质手绘边框
- **边框**：棕色木纹质感，粗度约 6-8px，带轻微不规则（模拟手绘笔触）
- **木纹**：使用 CSS `border-image` 或 box-shadow 多层叠加模拟木纹纹理，浅棕色到中棕色渐变

### 画面
- 显示真实摄像头画面（通过 `getUserMedia` 获取）
- 画面在"窗户"内，边缘有柔和的暗角（vignette）效果
- 背景视觉轻微虚化（CSS `backdrop-filter: blur()` 或用 canvas 处理）

### 装饰
- 窗户四角漂浮少量小星星（CSS animation，`@keyframes` 浮动 + 闪烁）
- 柔和暖光从窗户边缘向内散射（box-shadow 暖色光晕）
- 星星大小 6-12px，颜色星光金（#F5D76E 系列），透明度 0.3-0.6，随机延迟动画

## 二、中间聊天区（25%）

### 背景
- 浅米色羊皮纸质感：`#FAF3E3` 到 `#F5E6CC` 渐变
- 轻微纹理：CSS `background-image` 叠加噪点 pattern（SVG 或 base64 微小噪点）
- 圆角木框包裹，与摄像头区的木框风格一致

### 消息气泡

**小星的气泡**：
- 背景色：暖奶油白 `#FFF8EE`，边框 1px 蜂蜜金 `#E8C56D`
- 左侧带⭐小装饰（悬浮在气泡左上角）
- 气泡有微弱发光（box-shadow: 0 0 8px rgba(232, 197, 109, 0.3)）
- 圆角 16px，左侧底部略方（形成对话气泡感）

**用户的气泡**：
- 背景色：浅木棕 `#E8D5B7`，简洁无边框
- 圆角 16px，右侧底部略方
- 无装饰，保持自然简洁

### 字体
- **像素风**：使用像素字体（如 "Press Start 2P" 或更柔和的 "Silkscreen"、"DotGothic16"）
- 中文像素字体备选：站酷像素体、或使用 CSS `font-smooth: never` + `image-rendering: pixelated` 辅助
- 字号：小星消息 16-18px，用户消息 14-16px

### 小星角色（悬浮在聊天区左侧）

**位置**：聊天区左侧，固定位置，不随消息滚动

**外观规格**：
| 属性 | 规格 |
|------|------|
| 比例 | 二到三头身 Q 版（头:身 ≈ 1:2.5） |
| 风格 | 中性风，不偏男/女，温暖可爱 |
| 发型 | 柔软短发，颜色奶油白/浅银 |
| 发饰 | ⭐ 星星发夹（右侧） |
| 眼睛 | 琥珀色或银河色（深棕到紫渐变），大而圆 |
| 服装 | 星月斗篷（深蓝到紫渐变，带星星图案），内搭奶油白/鼠尾草绿 |
| 主色调 | 奶油白 #FFF8EE、浅木棕 #D4A574、蜂蜜金 #E8C56D、鼠尾草绿 #9CAF88 |
| 轮廓 | 带发光轮廓（soft glow，box-shadow 或 filter: drop-shadow） |
| 周围 | 身体周围 3-5 颗漂浮小星星 |

**动画规格**：
| 动画 | 触发方式 | 描述 |
|------|---------|------|
| 眨眼 | 循环，每 3-6 秒 | 2 帧动画，快速闭合再睁开（约 200ms） |
| 点头 | 随机，每 8-15 秒 | 头部轻微上下移动，约 300ms |
| 挥手 | 用户发送消息时 | 右手抬起左右摆动，约 600ms |
| 好奇看向 | 检测到摄像头画面变化时（后续）或用定时模拟 | 头部轻微倾斜，眼睛睁大 |

**实现方案**：CSS Sprite 动画 或 Lottie JSON 动画。推荐 Lottie（设计师友好，后续可精细迭代）。

## 三、底部工具栏（20%）

### 底座
- 木质底座，浅棕色，圆角（上圆角 16px，下圆角 0 贴合窗口底部）
- 与摄像头区木框同色系，颜色略深（形成视觉重心下沉）
- 轻微木纹纹理

### 按钮布局

```
[ 文本输入框...                  ] [🎤] [📞] [📷] [⚙️] [⭐]
```

| 按钮 | 图标 | 功能说明 |
|------|------|---------|
| 文本输入框 | — | 占工具栏约 60% 宽度，圆角 12px，羊皮纸色背景，像素字体 placeholder："跟我说点什么吧…" |
| 🎤 麦克风 | 手绘风格麦克风 | 按住说话 / 语音输入（后续接入 STT） |
| 📞 视频通话 | 手绘风格电话/摄像头 | 挂断/重连（核心交互） |
| 📷 拍照 | 手绘风格相机 | 截取当前摄像头画面（后续功能） |
| ⚙️ 设置 | 手绘风格齿轮 | 打开设置面板（音量、角色外观等） |
| ⭐ 收藏 | 手绘风格星星 | 收藏当前对话 / 精彩时刻 |

### 按钮规格
- 尺寸：40×40px（桌面端）
- Hover：轻微放大 + 发光（scale: 1.1, box-shadow 暖光）
- 点击：轻微缩小（scale: 0.95）
- 图标：SVG 或 CSS 手绘风格，线宽不均匀（模拟手绘）
- 颜色：深棕色 `#8B6914`，与木框协调

## 四、背景与环境

### 场景
小木屋内景，弱化虚化处理：

```
窗外植物(模糊)   📚书架(模糊)   挂画(模糊)
    ┌──────────────────────────┐
    │     [摄像头区 - 窗户]     │
    │                          │
    ├──────────────────────────┤
    │  ⭐小星 │ [对话区]       │
    ├──────────────────────────┤
    │     [工具栏 - 木底座]     │
    └──────────────────────────┘
  🪴小盆栽     💡夜灯      🪴小盆栽
```

### 环境元素（弱化、不抢主体）

| 元素 | 位置 | 处理 |
|------|------|------|
| 书架 | 左侧边缘外 | 模糊，暖棕色，隐约可见书本轮廓 |
| 小盆栽 | 左下角、右下角 | 简单植物剪影，鼠尾草绿 |
| 窗外景色 | 摄像头区窗户两侧隐约可见 | 模糊的绿色植物、阳光 |
| 挂画 | 右上角 | 简单风景/星星图案，模糊 |
| 夜灯 | 右下角 | 柔和暖光，小型灯台剪影 |

### 整体氛围
- 主光源来自左上方（模拟窗外自然光）
- 夜灯在右下补充暖光
- 整体柔光滤镜（CSS `::after` 伪元素，半透明暖色遮罩）

### 主色

|------|------|------|
| 奶油白 | `#FFF8EE` | 小星皮肤、气泡背景 |
| 浅木棕 | `#D4A574` | 木框边框、工具栏底座 |

| 色名 | 色值 | 用途 |
|------|------|------|

- ❌ 赛博风霓虹色

## 六、成功标准

- [ ] 页面垂直三层布局正确（55/25/20），窗口缩放时比例稳定
- [ ] 摄像头画面通过 `getUserMedia` 实时显示在"窗户"内
- [ ] 木框边框有手绘质感（非机械直线），圆角自然
- [ ] 小星角色渲染在聊天区左侧，包含 CSS/Lottie 循环动画（至少眨眼 + 漂浮）
- [ ] 对话气泡有两种样式（小星 vs 用户），视觉区分明确
- [ ] 工具栏 5 个按钮 + 输入框，布局正确，hover/click 有反馈
- [ ] 整体色调符合色彩体系规范，无禁止色
- [ ] 背景环境元素弱化不抢主体
- [ ] Electron 窗口启动后 2 秒内完成渲染
- [ ] 用 3 轮模拟对话验证聊天区滚动和气泡渲染

## 七、Scope 边界

**做什么**：
- Electron + React 桌面应用骨架
- 完整三层界面布局（摄像头/聊天/工具栏）
- 小星角色（静态 + 基础动画：眨眼、漂浮）
- 摄像头实时画面在窗户内显示
- 模拟对话数据验证 UI

**不做什么**：
- 不接入真实 AI（模拟数据即可）
- 不接入麦克风（按钮摆在那里，后端不做）
- 不实现设置面板（按钮摆在那里，点击无响应）
- 不做 Electron 打包（开发模式跑通即可）
- 不做小星角色的精细动画（Lottie 文件后续由设计师提供，先用 CSS 简易版）
- 不做实际的语音/视频通话功能
